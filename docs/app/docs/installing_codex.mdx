## Install Codex

Select your OS below for the steps to install Codex.

<Tabs>
<TabItem value="linux" label="Linux">

Run the following install script as a *non-root user* to install the latest version of Codex:

```bash
curl -fsSL https://raw.githubusercontent.com/khulnasoft/CodeX/master/codex | bash
```

Codex requires the [Nix Package Manager](https://nixos.org/download.html). If Nix is not detected when running a command, Codex will install it for you in single-user mode for Linux. Don't worry: You can use Codex without needing to learn the Nix Language.

If you would like to install Nix yourself, we recommend the [Determinate Nix Installer](https://determinate.systems/posts/determinate-nix-installer/).

</TabItem>
<TabItem value="macos" label="MacOS">

Run the following install script to install the latest version of Codex:

```bash
curl -fsSL https://raw.githubusercontent.com/khulnasoft/CodeX/master/codex | bash
```

Codex requires the [Nix Package Manager](https://nixos.org/download.html). If Nix is not detected when running a command, Codex will install it in multi-user mode for macOS. Don't worry: You can use Codex without needing to learn the Nix Language.

If you would like to install Nix yourself, we recommend the [Determinate Nix Installer](https://determinate.systems/posts/determinate-nix-installer).

</TabItem>

<TabItem value="wsl" label="Windows/WSL2">

You can use Codex on a Windows machine using [**Windows Subsystem for Linux 2**](https://learn.microsoft.com/en-us/windows/wsl/install). 

<details>
<summary>Installing WSL2</summary>

To install WSL2 with the default Ubuntu distribution, open Powershell or Windows Command Prompt as an administrator, and run:

```bash
wsl --install
```

If WSL2 is already installed, you can install Ubuntu by running

```bash
wsl --install -d Ubuntu
```
If you are running an older version of Windows, you may need to follow the [manual installation steps](https://learn.microsoft.com/en-us/windows/wsl/install-manual) to enable virtualization and WSL2 on your system. See the [official docs](https://learn.microsoft.com/en-us/windows/wsl/install) for more details

</details>

Run the following script in your WSL2 terminal as a *non-root user* to install Codex.

```bash
curl -fsSL https://raw.githubusercontent.com/khulnasoft/CodeX/master/codex | bash
```

Codex requires the [Nix Package Manager](https://nixos.org/download/). If Nix is not detected on your machine when running a command, Codex will automatically install it in single user mode for WSL2. Don't worry: You can use Codex without needing to learn the Nix Language.

</TabItem>

<TabItem value="nix" label="NixOS/Nixpkg">

Codex is available through the [**Nix Package Manager**](https://search.nixos.org/packages?channel=unstable&show=codex&from=0&size=50&sort=relevance&type=packages&query=codex). 

To install on NixOS:

```bash
nix-env -iA nixos.codex
```

To install on a non NixOS:

```bash
nix-env -iA nixpkgs.codex
```

or 

```bash
nix profile install nixpkgs#codex
```

</TabItem>
</Tabs>

## Updating Codex

Codex will notify you when a new version is available. To update to the latest released version of Codex, you can run `codex version update`. 

If you installed Codex with Nix, you can update Codex via Nix using `nix-env -u codex`, or `nix profile upgrade`.

You can find release notes and details on the [Releases page](https://github.com/khulnasoft/codex/releases) of the Codex Github repo.

## Rolling Back or Pinning a Specific Version of Codex

You can rollback or pin the version of Codex on your system using the `CODEX_USE_VERSION` environment variable. For example, to use version 0.8.0: 

```bash
export CODEX_USE_VERSION=0.8.0
```

Setting this variable will use the specified version of Codex even if a newer version has been installed on your machine. 

If you installed Codex with Nixpkgs, you will need to pin Codex in your profile or Nix configuration. You can find the Nixpkg commits for previous versions of Codex on [Nixhub](https://www.nixhub.io/packages/codex).
